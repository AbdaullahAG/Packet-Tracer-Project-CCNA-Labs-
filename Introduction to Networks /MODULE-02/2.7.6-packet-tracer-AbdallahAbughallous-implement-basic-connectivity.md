# Lab Report: Packet Tracer - Implement Basic Connectivity

## 1. Overview
This lab exercise demonstrates the fundamental steps required to establish basic network connectivity. The tasks include the initial configuration of Cisco Catalyst switches, assigning IP addresses to end devices (PCs), and configuring the Switch Management Interface (SVI) to allow remote management and network testing.

## 3. Lab Objectives & Task Analysis

### Part 1: Perform a Basic Configuration on S1 and S2
**Objective:** Secure the device and identify it on the network.

* **Password Verification:** To verify passwords, use the `show running-config` command to check the configuration file, or exit to the user prompt to test the login requirement.
* **Saving Changes:** The command `copy running-config startup-config` was used to migrate the configuration from volatile RAM to non-volatile NVRAM.

### Part 2: Configure the PCs
**Objective:** Assign logical identities to end-user workstations.

* **Connectivity Check:** Initial pings from PC1 to S1 were **unsuccessful** at this stage. 
* **Reason:** Although the PC was configured, the switch management interface (VLAN 1) had not yet been assigned an IP address, meaning the switch could not yet participate in IP-based communication.

### Part 3: Configure the Switch Management Interface
**Objective:** Enable the switch for remote management and layer 3 communication.

* **Management IP Necessity:** Switches require an IP address so that administrators can manage them remotely (via SSH/Telnet) and to allow the switch to respond to diagnostic tools like `ping`.
* **Interface Activation:** The `no shutdown` command is critical because VLAN interfaces are often administratively disabled by default.

## 4. Configuration Procedures

### Switch S1 & S2 Basic Setup (CLI)
The following commands were applied to both S1 and S2 (with respective hostname changes):

```bash
# Example Configuration for S1
S1> enable
S1# configure terminal
S1(config)# hostname S1
S1(config)# line console 0
S1(config-line)# password cisco
S1(config-line)# login
S1(config-line)# exit
S1(config)# enable password class
S1(config)# banner motd "Authorized access only. Violators will be prosecuted."
S1(config)# interface vlan 1
S1(config-if)# ip address 192.168.1.253 255.255.255.0
S1(config-if)# no shutdown
S1(config-if)# end
S1# copy running-config startup-config
