# Lab Report: Packet Tracer - Basic Switch and End Device Configuration

## 1. Objective
The purpose of this lab was to perform initial configurations on two Cisco Catalyst switches (ASw-1 and ASw-2) and 
two end devices (User-01 and User-02). Key objectives included:
* Configuring hostnames and securing access via passwords.
* Implementing service password encryption and MOTD banners.
* Configuring the Switch Management Interface (SVI) for remote access.
* Verifying end-to-end connectivity across the LAN.

## 3. Configuration Requirements & Implementation

### A. Switch Security & Identity
The switches were configured to prevent unauthorized access through the following security layers:
* **Hostnames:** Set to `ASw-1` and `ASw-2` for network identification.
* **Console Security:** Protected line console 0 with the password `xAw6k`.
* **Remote Access:** Protected VTY lines 0-15 with the password `xAw6k`.
* **Privileged Mode:** Secured using a strong encrypted secret: `6EBUp`.
* **Encryption:** Enabled `service password-encryption` to ensure all clear-text passwords are encrypted in the configuration file.
* **Banner:** Configured a Message of the Day (MOTD) to provide a legal warning to unauthorized users.

### B. Logical Addressing
* **Management Interface:** VLAN 1 was assigned the respective IP addresses from the addressing table on both switches and activated using the `no shutdown` command.
* **End Devices:** PC User-01 and User-02 were configured with static IP addresses and the standard Class C subnet mask (`255.255.255.0`).

## 4. Configuration Procedures (CLI Example)
The following commands represent the standard configuration applied to the switches:

```bash
# Example for Switch ASw-1
Switch> enable
Switch# configure terminal
Switch(config)# hostname ASw-1
Switch(config)# enable secret 6EBUp
Switch(config)# line console 0
Switch(config-line)# password xAw6k
Switch(config-line)# login
Switch(config-line)# exit
Switch(config)# line vty 0 15
Switch(config-line)# password xAw6k
Switch(config-line)# login
Switch(config-line)# exit
Switch(config)# service password-encryption
Switch(config)# banner motd "Authorized Access Only!"
Switch(config)# interface vlan 1
Switch(config-if)# ip address 10.10.10.100 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# do copy running-config startup-config
5. Verification and Connectivity
To ensure the lab requirements were met, the following verification steps were performed:

Show Commands: Used show ip interface brief to verify that VLAN 1 interfaces were "Up/Up".

Configuration Check: Used show running-config to verify that all passwords were encrypted and the banner was active.

Connectivity Testing: Performed a successful ping from User-01 to User-02 and from the PCs to the Switch Management Interfaces.

6. Conclusion
All configuration tasks were completed successfully. The switches are now secured with encrypted passwords, and end-to-end connectivity has been established between all devices in the 10.10.10.0/24 subnet. The final configuration was saved to NVRAM to ensure persistence after a reboot.
