# 📝 Lab Report: Basic Switch and End Device Configuration (Physical Mode)

---

## 🛡️ 1. Overview & Objectives
The primary goal of this lab was to demonstrate the ability to build a physical network topology, configure basic switch settings, and establish connectivity between multiple hosts.

**Key Objectives:**
* **Physical Layer:** Set up the network topology using correct cabling (Straight-through vs. Cross-over).
* **Host Configuration:** Configure static IP addressing on PC hosts.
* **Switch Management:** Initialize Cisco IOS switches with security settings and SVI (Switch Virtual Interface).
* **Verification:** Confirm end-to-end connectivity using standard IOS diagnostic tools.

## 🛠️ 3. Implementation Steps

### Part 1: Physical Topology Setup
In this phase, devices were moved from the shelf to the rack and cabled correctly:
* **S1 (F0/1) ↔ S2 (F0/1):** Copper Cross-Over cable (Switch to Switch).
* **S1 (F0/6) ↔ PC-A:** Copper Straight-Through cable.
* **S2 (F0/18) ↔ PC-B:** Copper Straight-Through cable.

### Part 2: Host Configuration
Static IP parameters were applied to PC-A and PC-B. Connectivity was initially tested between the two PCs via the Command Prompt using the `ping` command.

### Part 3: Basic Switch Configuration
The following core configurations were applied to both **S1** and **S2** via Console access:
1.  **Identity:** Hostname assignment.
2.  **Security:**
    * **Privileged EXEC:** `class` (secret).
    * **Console Access:** `cisco` (password).
3.  **Management Interface:** VLAN 1 IP addressing.
4.  **Security Awareness:** Configure MOTD (Message of the Day) banner.
5.  **Persistence:** Saving the `running-config` to `startup-config` (NVRAM).

## 💻 4. Configuration Procedures (CLI Example)
Applied commands on **Switch S1**:

```bash
Switch> enable
Switch# configure terminal
Switch(config)# hostname S1
Switch(config)# enable secret class
Switch(config)# line console 0
Switch(config-line)# password cisco
Switch(config-line)# login
Switch(config-line)# exit
Switch(config)# banner motd "UNAUTHORIZED ACCESS IS PROHIBITED!"
Switch(config)# interface vlan 1
Switch(config-if)# ip address 192.168.1.1 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# end
Switch# copy running-config startup-config
