# Packet Tracer - Troubleshoot Default Gateway Issues

## 📋 Project Overview
This lab focuses on the process of verifying network documentation, isolating connectivity issues, and resolving **Default Gateway** misconfigurations. End-to-end connectivity is only possible when all hosts, switches, and routers are correctly addressed.

## 🔍 Part 2: Troubleshooting Documentation
Use this log to track identified issues and their corresponding solutions during the troubleshooting process.

| Test | Successful? | Issue Found | Solution | Verified |
| :--- | :---: | :--- | :--- | :---: |
| PC1 to PC2 | **No** | PC1 has an incorrect IP address. | Change PC1 IP to `192.168.10.10`. | ✅ |
| PC1 to S1 | **No** | S1 is missing the default gateway. | Configure `ip default-gateway` on S1. | ✅ |
| PC3 to PC4 | **No** | PC4 gateway is missing or wrong. | Correct Gateway to `192.168.11.1`. | ✅ |
| PC1 to PC4 | **No** | Remote gateway on S2/PCs is wrong. | Update S2 and PC gateway settings. | ✅ |

## 🚀 Implementation Steps

### 1. Switch Configuration (S1 & S2)
To allow remote management and cross-network communication, the switches must have a gateway configured.

**For S1:**
```bash
S1> enable
S1# configure terminal
S1(config)# ip default-gateway 192.168.10.1
S1(config)# exit
S1# copy run start
For S2:
```
```bash
S2> enable
S2# configure terminal
S2(config)# ip default-gateway 192.168.11.1
S2(config)# exit
S2# copy run start
```
2. PC Configuration
Click on the PC, go to Desktop > IP Configuration.

PC1: Update IP to 192.168.10.10 and Gateway to 192.168.10.1.

PC4: Update Gateway to 192.168.11.1.

### ✅ Final Verification
Note: A successful troubleshoot is only complete when End-to-End Connectivity is verified.

Open the Command Prompt on PC1.

Run: ping 192.168.11.11 (PC4).

The test is successful if you receive 4 replies.
