## 2. Router Configuration (Floor14)
Commands to configure the hostname, security, and interfaces on the router:

```text
enable
configure terminal
hostname Floor14

# Security and Passwords
enable secret class
line con 0
 password cisco
 login
 exit
line vty 0 4
 password cisco
 login
 exit
service password-encryption

# Banner
banner motd "Authorized Access Only!"

# Interface G0/0 Configuration
interface g0/0
 description Connection to Room-145 LAN
 ip address 172.14.5.1 255.255.255.0
 ipv6 address 2001:DB8:CAFE:1::1/64
 ipv6 address fe80::1 link-local
 no shutdown
 exit

# Interface G0/1 Configuration
interface g0/1
 description Connection to Room-146 LAN
 ip address 172.14.10.1 255.255.255.0
 ipv6 address 2001:DB8:CAFE:2::1/64
 ipv6 address fe80::2 link-local
 no shutdown
 exit

# Enable IPv6 Routing
ipv6 unicast-routing

# Save Configuration
do write

```
## 3. Switch Configuration (Room-146)
Commands to configure the switch management interface and gateway:

```Plaintext
enable
configure terminal
hostname Room-146

# Security and Passwords
enable secret class
line con 0
 password cisco
 login
 exit
line vty 0 15
 password cisco
 login
 exit
service password-encryption

# Interface VLAN 1 Configuration
interface vlan 1
 description Management Interface
 ip address 172.14.10.35 255.255.255.0
 no shutdown
 exit

# Default Gateway
ip default-gateway 172.14.10.1

# Save Configuration
do write
```

## 5. Verification & Troubleshooting
Connectivity Test: From Manager-A, use the command prompt to ping Manager-B (IPv4 and IPv6).

Status Check: Use show ip interface brief and show ipv6 interface brief on the router to ensure all interfaces are "Up/Up".

Issue Found: The switches were not reachable initially because the default gateway was missing in the switch configuration.

Solution: Applied ip default-gateway 172.14.10.1 on Room-146.
