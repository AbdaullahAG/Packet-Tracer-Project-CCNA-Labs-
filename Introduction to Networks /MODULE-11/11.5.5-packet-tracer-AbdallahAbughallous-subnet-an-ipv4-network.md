# Packet Tracer – Subnet an IPv4 Network

This repository contains the solution and documentation for the IPv4 Subnetting activity. The project involves designing a subnetting scheme, configuring network devices (Router, Switches, PCs), and verifying connectivity.

---

## 📋 Objectives
* **Part 1:** Design an IPv4 Network Subnetting Scheme.
* **Part 2:** Configure the Devices (Router, Switches, and PCs).
* **Part 3:** Test and Troubleshoot the Network.

---

## 🛠️ Part 1: Subnetting Design

### 1. Requirements Analysis
* **Network Address:** `192.168.0.0/24`
* **LAN-A Requirement:** Minimum 50 hosts.
* **LAN-B Requirement:** Minimum 40 hosts.
* **Future Expansion:** At least 2 additional unused subnets.
* **Constraint:** All subnets must use the same mask length.

### 2. Subnet Mask Selection
To support at least 50 hosts and 4 subnets, the **/26** prefix was chosen.

| Prefix | Subnet Mask | Subnets ($2^s$) | Hosts ($2^h-2$) | Status |
| :--- | :--- | :--- | :--- | :--- |
| **/25** | 255.255.255.128 | 2 | 126 | ❌ Not enough subnets |
| **/26** | **255.255.255.192** | **4** | **62** | ✅ **Optimal Choice** |
| **/27** | 255.255.255.224 | 8 | 30 | ❌ Not enough hosts |

### 3. Derived Subnets Table
| Subnet | Network Address | Prefix | Subnet Mask |
| :--- | :--- | :--- | :--- |

### CustomerRouter Configuration
```BASH

configure terminal
hostname CustomerRouter
enable secret Class123
line console 0
 password Cisco123

exit

 ip address 192.168.0.1 255.255.255.192
 no shutdown

interface g0/1

exit
```
