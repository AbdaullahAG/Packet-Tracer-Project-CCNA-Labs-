### Introduction
In this activity, we are given the network address **192.168.72.0/24** to design a **VLSM** addressing scheme based on the host requirements for each LAN and the WAN link between the two buildings.

---

### Part 1: Examine the Network Requirements

**Step 1: Determine the number of subnets needed**  
Number of subnets required = **5**  
(4 LANs for the switches + 1 WAN link between Building1 and Building2)

**Step 2: Determine the subnet mask information for each subnet**

**a.** ASW-1 LAN (requires 7 host IP addresses)  
- Subnet Mask: **/28** (255.255.255.240)  
- Usable Host Addresses: **14**

**b.** ASW-2 LAN (requires 15 host IP addresses)  
- Subnet Mask: **/27** (255.255.255.224)  
- Usable Host Addresses: **30**

**c.** ASW-3 LAN (requires 29 host IP addresses)  
- Subnet Mask: **/27** (255.255.255.224)  
- Usable Host Addresses: **30**

**d.** ASW-4 LAN (requires 58 host IP addresses)  
- Subnet Mask: **/26** (255.255.255.192)  
- Usable Host Addresses: **62**

**e.** Connection between Building1 and Building2 (WAN Link)  
- Subnet Mask: **/30** (255.255.255.252)  
- Usable Host Addresses: **2**

---

### Part 2: Design the VLSM Addressing Scheme

**Step 1:** The 192.168.72.0/24 network was divided starting with the largest subnet first (ASW-4 LAN), then in descending order.

**Step 2: Subnet Table**

| Subnet Description                  | Number of Hosts Needed | Network Address/CIDR     | First Usable Host Address | Broadcast Address     |
|-------------------------------------|------------------------|--------------------------|---------------------------|-----------------------|
| ASW-4 LAN                          | 58                     | 192.168.72.0/26         | 192.168.72.1             | 192.168.72.63        |
| ASW-3 LAN                          | 29                     | 192.168.72.64/27        | 192.168.72.65            | 192.168.72.95        |
| ASW-2 LAN                          | 15                     | 192.168.72.96/27        | 192.168.72.97            | 192.168.72.127       |
| ASW-1 LAN                          | 7                      | 192.168.72.128/28       | 192.168.72.129           | 192.168.72.143       |
| WAN Link (Building1 – Building2)   | 2                      | 192.168.72.144/30       | 192.168.72.145           | 192.168.72.147       |

**Step 3: Document the addressing scheme**

- **Building1 Router** (uses First Usable address):  
  - G0/0 → 192.168.72.129/28 (ASW-1 LAN)  
  - G0/1 → 192.168.72.97/27  (ASW-2 LAN)  
  - S0/0/0 → 192.168.72.145/30 (WAN Link)

- **Building2 Router**:  
  - G0/0 and G0/1 → First Usable in their respective LANs  
  - S0/0/0 → 192.168.72.146/30 (Last Usable on WAN)

- **Switches** (use Second Usable address):  
  - ASW-1 VLAN 1 → 192.168.72.130/28  
  - ASW-2 VLAN 1 → 192.168.72.98/27  
  - ASW-3 VLAN 1 → 192.168.72.66/27  
  - ASW-4 VLAN 1 → 192.168.72.2/26  

- **Hosts** (use Last Usable address):  
  - Host-A → 192.168.72.142/28  
  - Host-B → 192.168.72.126/27  
  - Host-C → 192.168.72.94/27  
  - Host-D → 192.168.72.62/26  

---

### Part 3: Assign IP Addresses to Devices and Verify Connectivity

**Step 1:** Configured IP addressing on Building1 router LAN interfaces:

```bash
interface GigabitEthernet0/0
 ip address 192.168.72.129 255.255.255.240
 no shutdown

interface GigabitEthernet0/1
 ip address 192.168.72.97 255.255.255.224
 no shutdown
 ```
**Step 2:** Configured ASW-3 switch including default gateway:
```Bash
interface Vlan 1
 ip address 192.168.72.66 255.255.255.224
 no shutdown

ip default-gateway 192.168.72.65
```
**Step 3:** Configured Host-D:

IP Address: 192.168.72.62
Subnet Mask: 255.255.255.192
Default Gateway: 192.168.72.1

**Step 4:** Verify Connectivity
Connectivity was successfully verified using ping commands from:

Building1 Router
ASW-3 Switch
Host-D

All IP addresses listed in the Addressing Table were reachable with 100% success rate.

### Conclusion
The VLSM addressing scheme was successfully designed and implemented by allocating subnets starting from the largest requirement to the smallest to avoid address overlap. IP addresses were assigned following Cisco best practices (First Usable for routers, Second Usable for switches, and Last Usable for hosts).
Full end-to-end connectivity was achieved and verified.
