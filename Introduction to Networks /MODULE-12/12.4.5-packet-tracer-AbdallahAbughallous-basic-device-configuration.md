# Packet Tracer - Basic Device Configuration

## Project Overview
This project demonstrates the configuration of a Cisco router (Floor14) and a switch (Room-146) to establish connectivity between two LANs.
It covers basic security, IPv4/IPv6 addressing, and connectivity verification.

## 1. Objectives
* Complete the network documentation.
* Perform basic device configurations (Hostname, Passwords, Banners).
* Configure IPv4 and IPv6 addressing for network devices and hosts.
* Verify end-to-end connectivity and troubleshoot issues.

## 3. Configuration Scripts

### Floor14 Router Configuration
```bash
enable
configure terminal
hostname Floor14
enable secret class
line con 0
 password cisco
 login
line vty 0 4
 password cisco
 login
exit
service password-encryption
banner motd "Authorized Access Only!"
ipv6 unicast-routing

interface g0/0
 description Link to LAN A
 ip address 128.107.20.1 255.255.255.0
 ipv6 address 2001:DB8:A::1/64
 ipv6 address fe80::3 link-local
 no shutdown

interface g0/1
 description Link to LAN B
 ip address 128.107.30.1 255.255.255.0
 ipv6 address 2001:DB8:B::1/64
 ipv6 address fe80::1 link-local
 no shutdown
exit
copy running-config startup-config
```
### Room-146 Switch Configuration
```Bash
enable
configure terminal
hostname Room-146
enable secret class
line con 0
 password cisco
 login
exit
service password-encryption
banner motd "Authorized Access Only!"

interface vlan 1
 description Management Interface
 ip address 128.107.30.15 255.255.255.0
 no shutdown
exit
ip default-gateway 128.107.30.1
copy running-config startup-config
```
## 4. Verification and Troubleshooting
IPv4 Connectivity: Verified using ping from Manager-A to Manager-B via the IPv4 gateway.

IPv6 Connectivity: Verified using ping from Manager-A to Manager-B via the IPv6 Global Unicast Addresses.

Security Check: Verified that all passwords are encrypted in the configuration file and the banner is displayed upon login.

5. Conclusion
Full end-to-end connectivity was successfully established. The router was configured to handle inter-VLAN routing for both IPv4 and IPv6 traffic, and the switch was configured with a management IP for remote access.
