# Lab Report: Implement a Subnetted IPv6 Addressing Scheme

## 1. Objective
The goal of this lab was to design and implement an IPv6 subnetting scheme for a network with four LANs and a point-to-point link between two routers (R1 and R2).

## 2. IPv6 Subnetting Scheme
Starting from the base subnet `2001:db8:acad:00c8::/64`, the addresses were incremented consecutively for each required network:

| Network Description | Subnet Address |
| :--- | :--- |
| **R1 G0/0 LAN** | `2001:db8:acad:00c8::/64` |
| **R1 G0/1 LAN** | `2001:db8:acad:00c9::/64` |
| **R2 G0/0 LAN** | `2001:db8:acad:00ca::/64` |
| **R2 G0/1 LAN** | `2001:db8:acad:00cb::/64` |
| **R1 to R2 Link** | `2001:db8:acad:00cc::/64` |

---

## 3. Addressing Implementation
### Router Configuration Logic:
- **LAN Interfaces:** Assigned the **first** address (`::1`) from their respective subnets.
- **Link-Local:** Manually assigned as `fe80::1` for R1 and `fe80::2` for R2.
- **Serial Link:** R1 was assigned `::1` and R2 was assigned `::2` from the `00cc` subnet.

### Host Configuration:
- All PCs (PC1-PC4) were set to **SLAAC (Auto-Config)** to automatically receive their IPv6 addresses and Gateway information.

---

## 4. Configuration Steps
The following commands were applied to the routers:
1.  **Enable IPv6 Routing:** Using the command `ipv6 unicast-routing`.
2.  **Interface Configuration:** Assigning global unicast addresses (GUA) and link-local addresses.
3.  **Activation:** Ensuring all interfaces were enabled using the `no shutdown` command.

---

## 5. Verification & Results
- **Interface Status:** Verified that all interfaces are "Up/Up" using `show ipv6 interface brief`.
- **Routing Table:** Confirmed that R1 and R2 can see the subnets via `show ipv6 route`.
- **Connectivity:** Successful `ping` tests were performed between PCs across the different subnets, confirming that routing is functional.

## 6. Conclusion
The IPv6 addressing scheme was successfully implemented. By using consecutive hex increments, the subnets were organized logically, and the use of SLAAC simplified the host configuration process.
