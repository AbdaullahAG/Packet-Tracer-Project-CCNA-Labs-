# Lab Report: Configuring Manual IPv6 Addressing

## 1. Objective
The primary objective of this lab was to manually configure IPv6 global unicast addresses (GUA) and link-local addresses on a router, a switch, and host PCs. Additionally, it aimed to verify connectivity and understand the role of Stateless Address Autoconfiguration (SLAAC).

## 2. Addressing Scheme
The network was divided into two main subnets. The first subnet was assigned to the LAN connected to G0/0/0, and the second to the LAN connected to G0/0/1.

| Device | Interface | IPv6 GUA Address | Link-Local Address |
| :--- | :--- | :--- | :--- |
| **R1** | G0/0/0 | `2001:db8:acad:1::1/64` | `fe80::1` |
| **R1** | G0/0/1 | `2001:db8:acad:2::1/64` | `fe80::1` |
| **S1** | VLAN 1 | `2001:db8:acad:1::b/64` | `fe80::2` |
| **PC-A** | NIC | `2001:db8:acad:1::3/64` | `fe80::1` (Gateway) |
| **PC-B** | NIC | `2001:db8:acad:2::3/64` | `fe80::1` (Gateway) |

---

## 3. Configuration Steps Summary

### Router Configuration (R1):
* **IPv6 Unicast Routing:** Enabled to allow the router to forward IPv6 packets and send Router Advertisement (RA) messages.
* **Interface Assignment:** Configured Global Unicast Addresses and set the Link-Local address to `fe80::1` for management simplicity.
* **Multicast Groups:** Verified that R1 joined the `ff02::2` (All-Routers) multicast group.

### Switch Configuration (S1):
* **Management Interface:** Configured the SVI (VLAN 1) with an IPv6 address to allow remote management within the IPv6 network.

### Host Configuration (PCs):
* **Static Configuration:** Assigned specific IPv6 addresses as required.
* **SLAAC:** Verified that PCs received additional IPv6 information automatically once routing was enabled on R1.

---

## 4. Technical Reflections & Findings
* **Link-Local Reuse:** The same link-local address (`fe80::1`) was used on multiple interfaces of R1 because its scope is limited to the local link (broadcast domain), preventing any addressing conflicts.
* **Subnet Identification:** For the address `2001:db8:acad::aaaa:1234/64` with a `/48` prefix, the Subnet ID is identified as `0000`.
* **Multicast Communication:** The implementation confirmed that IPv6 relies on multicast groups (like `ff02::1` and `ff02::2`) instead of traditional IPv4 broadcasting.

## 5. Connectivity Verification
* **Ping Tests:** Successful pings were executed from PC-A to the router's link-local address and to PC-B's global address.
* **Trace Route:** End-to-end connectivity was verified using `tracert`, showing the path through the R1 gateway.

## 6. Conclusion
The lab successfully demonstrated the manual implementation of IPv6. It highlighted the efficiency of SLAAC and the importance of correctly configuring link-local addresses for gateway functionality.
